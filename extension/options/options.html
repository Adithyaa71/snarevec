Profiles

A profile = backend + storage target + collection + chunking, bundled so the popup is one dropdown. Chunking and model choices are locked per collection — mixing them corrupts search quality.

Embedding backends

Where vectors get computed. For byte-compatible vectors across every machine, use fastembed with nomic-embed-text-v1.5 — same library, same model, everywhere.

Storage targets

Where finished vectors go: staged JSONL on this laptop, pushed over SSH to another machine, or straight into a Qdrant server. (Embedded/path-mode Qdrant needs the staged-file route + the import script.)

Queue

Captures that couldn't be delivered — dead backend, unreachable target — wait here. Nothing is lost.

Daemon settings

The extension only stores the daemon address and token. API keys and SSH credentials live in the daemon's config file on your disk, never in the browser.
